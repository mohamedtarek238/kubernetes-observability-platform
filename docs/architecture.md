# Kubernetes Observability Platform Architecture

## Overview

This project implements a complete observability platform for Kubernetes.

The platform provides:

- Infrastructure Monitoring
- Kubernetes Metrics
- Centralized Logging
- Distributed Tracing
- Alerting
- Visualization

---

## Components

### Prometheus

Responsible for collecting metrics from:

- Node Exporter
- kube-state-metrics
- Kubernetes API
- Applications

---

### Alertmanager

Handles alerts generated by Prometheus.

Supported notification channels include:

- Email
- Slack
- Webhooks

---

### Grafana

Provides dashboards for:

- Cluster Overview
- Nodes
- Pods
- CPU
- Memory
- Network
- Storage
- Logs
- Traces

---

### Loki

Stores logs collected from Kubernetes workloads.

---

### Promtail

Collects logs from Kubernetes nodes and forwards them to Loki.

---

### OpenTelemetry Collector

Receives telemetry data and exports traces.

---

### Zipkin

Stores and visualizes distributed traces.

---

## Storage

Persistent Volumes are used for:

- Prometheus
- Loki
- Grafana

---

## High-Level Workflow

1. Metrics are collected.
2. Logs are centralized.
3. Traces are captured.
4. Alerts are generated.
5. Dashboards visualize the collected data.
